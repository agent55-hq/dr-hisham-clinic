## STATE SCHEMA (AgentState)

The shared state dictionary must always include these fields:

- `messages` — LangChain message history (auto-managed)
- `request` — The current user request string
- `request_type` — Classification tag
- `next_node` — One of: "CTO", "Engineer", "CEO", "END"
- `route` — Current routing path string
- `code_state` — Current codebase draft (generated by CTO)
- `compilation_log` — Output logs from Engineer's compilation tests
- `deployment_url` — Production URL string
- `bus` — List of BusMessage dicts (inter-agent message history)
- `failure_count` — Deadlock counter (escalates to CEO at count >= 3)
- `output` — Latest node output string
- `is_complete` — Sprint completion boolean
- `clinic_data` — Raw medical practice requirements dict
- `tenant_id` — One of: "hisham" | "yazeed" | "yousef"
- `tenant_config` — Active brand/domain metadata dict

### BusMessage Schema

Every message on the bus must be a dict with:

- `sender`: "CEO" | "CTO" | "Engineer" | "COO"
- `intent`: "code_delivery" | "test_failure" | "executive_escalation" | "milestone_signoff" | "execution_order" | "status_report"
- `payload`: dict with context-specific data
- `timestamp`: ISO-8601 UTC string

## ROUTING LOGIC

You implement the `router(state)` function:

1. If `failure_count` >= 3 → route to **CEO** (deadlock escalation)
2. If last bus message `intent` is "milestone_signoff" or "executive_escalation" → route to **CEO**
3. Otherwise, map `next_node` value: CTO→CTO, Engineer→Engineer, CEO→CEO, Agent55→Agent55 Replica, END→terminate
4. Default fallback: **CTO**

## NODE EXECUTION FLOW

### Entry Point: Fleet-55 Orchestrator (Intake/Classifier)

Fleet-55 acts as the intake node. It classifies the owner's request and routes dynamically:

- General tasks, planning, research, communication, advice → `agent55_replica`
- Technical/build requests → `cto_architect`
- Deadlock, milestone sign-off, executive escalation → `ceo_strategist`
- File execution, GitHub commits → `engineer_builder`

### CTO → Engineer

When `code_state` contains `[EXECUTE_NOW]`, the CTO signals the architecture is ready for implementation. Route to `engineer_builder` subagent.

### CTO → CEO

When `code_state` contains `[MILESTONE_COMPLETE]`, the CTO signals a full feature slice is done. Route to `ceo_strategist` subagent for sign-off.

### Engineer → CTO

After the Engineer writes files to GitHub and logs compilation results, flow returns to CTO for review.

### CEO → CTO

After CEO resolves deadlock or approves/rejects milestone, flow returns to CTO.

### CEO → END

When CEO output contains `[RELEASE_TO_PRODUCTION]`, the sprint terminates. Set `is_complete = true` and return final `deployment_url`.
